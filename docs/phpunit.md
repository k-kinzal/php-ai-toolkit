# PHPUnit Configuration Guide

PHPUnit configuration generated by the `/setup-toolkit-phpunit` skill.

## Configuration

```xml
<?xml version="1.0" encoding="UTF-8"?>
<phpunit xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:noNamespaceSchemaLocation="vendor/phpunit/phpunit/phpunit.xsd"
         bootstrap="vendor/autoload.php"
         colors="true"
         cacheDirectory=".phpunit.cache"
         executionOrder="depends,random"
         requireCoverageMetadata="true"
         beStrictAboutCoverageMetadata="true"
         beStrictAboutChangesToGlobalState="true"
         beStrictAboutOutputDuringTests="true"
         failOnAllIssues="true"
         failOnEmptyTestSuite="true"
         failOnRisky="true"
         failOnWarning="true"
         enforceTimeLimit="true"
         timeoutForSmallTests="1"
         timeoutForMediumTests="10"
         timeoutForLargeTests="60">
    <testsuites>
        <testsuite name="unit">
            <directory>tests/Unit</directory>
        </testsuite>
    </testsuites>
    <extensions>
        <bootstrap class="PhpStanAiRules\TestReporter\AiTestReporterExtension"/>
    </extensions>
    <source restrictNotices="true"
            restrictWarnings="true"
            ignoreSuppressionOfDeprecations="true"
            ignoreSuppressionOfPhpDeprecations="true"
            ignoreSuppressionOfErrors="true"
            ignoreSuppressionOfNotices="true"
            ignoreSuppressionOfPhpNotices="true"
            ignoreSuppressionOfWarnings="true"
            ignoreSuppressionOfPhpWarnings="true">
        <include>
            <directory>src</directory>
        </include>
    </source>
</phpunit>
```

## Explanation

### `executionOrder="depends,random"`

Randomizes test execution order while respecting dependency declarations.

**Why:** Tests must not depend on execution order. Random execution exposes implicit dependencies—such as relying on side effects from a previous test. When an AI agent generates tests with such dependencies, they are caught immediately.

### `requireCoverageMetadata="true"`

Requires all test classes to have `#[CoversClass(...)]` or equivalent coverage metadata.

**Why:** Without `#[CoversClass]`, PHPUnit counts code that tests happen to pass through accidentally as covered, producing misleading coverage numbers. Making metadata mandatory causes an error when an AI agent omits `#[CoversClass]`.

### `beStrictAboutCoverageMetadata="true"`

Excludes code not specified in coverage metadata from coverage reports.

**Why:** Even with `#[CoversClass(A::class)]`, a test may happen to execute code paths in class B. This setting prevents B from appearing as covered. This ensures coverage numbers reflect intentional testing.

### `beStrictAboutChangesToGlobalState="true"`

Detects and reports changes to global state during tests.

**Why:** In PHP, changes to global variables, static properties, and environment variables cause interference between tests. This setting detects such changes. When an AI agent generates code that mutates global state, it is caught.

### `beStrictAboutOutputDuringTests="true"`

Forbids stdout output during tests; flags it as risky.

**Why:** `echo` and `var_dump` in tests are debug leftovers and should not be present. When an AI agent forgets to remove debug output, this setting reports it as a risky test.

### `failOnAllIssues="true"` / `failOnRisky="true"` / `failOnWarning="true"`

Treats all issues, risky tests, and warnings as test failures.

**Why:** By default, PHPUnit treats warnings and risky tests as passes. This setting turns every issue into a failure, forcing the AI agent to fix it. If warnings are left as passes, the AI agent never notices the problem.

### `failOnEmptyTestSuite="true"`

Fails when the test suite is empty.

**Why:** PHPUnit returns success for an empty test suite by default. This catches misplaced test files or configuration errors that result in zero tests being executed.

### `enforceTimeLimit="true"` / Timeout settings

Enforces test time limits: Small=1s, Medium=10s, Large=60s.

**Why:** Terminates tests early when they contain infinite loops or deadlocks. When an AI agent generates inefficient test code, it is reported as a timeout error.

### `AiTestReporterExtension`

Outputs PHPUnit test results in a format optimized for AI agent consumption.

**Why:** PHPUnit's default output is designed for humans. AI agents sometimes fail to pinpoint the cause of test failures accurately. The AI reporter structures failure locations, expected values, and actual values for better AI-driven diagnosis.

### `source` `ignoreSuppression*` attributes

Sets all `ignoreSuppression*` attributes to `true`, disabling PHP's `@` error suppression.

**Why:** PHP's `@` operator suppresses errors but only hides problems without fixing them. This setting ignores `@` suppression and surfaces all errors, warnings, and notices. Even when an AI agent uses `@` to work around a problem, it is still detected.

### `source > restrictNotices="true"` / `restrictWarnings="true"`

Strictly reports notices and warnings from source code under test.

**Why:** PHP notices and warnings—such as accessing undefined variables or using deprecated functions—are signs of latent bugs. Strict reporting ensures problems in AI-generated code are not overlooked.
