# Deptrac Configuration

Deptrac configuration generated by the `/setup-toolkit-deptrac` skill.

## Purpose

Deptrac enforces dependency direction between architectural layers. It should describe the project's real boundaries, not a generic web stack. The same setup should work for web apps, CLI apps, workers, libraries, packages, and modular codebases after adapting layer names and collectors.

## Installation

Use Composer when the target project can satisfy Deptrac's runtime constraints:

```bash
composer require --dev deptrac/deptrac
```

Do not force installation with `--ignore-platform-reqs`. For libraries and developer tools that support older PHP versions, allow Composer to resolve different Deptrac versions per PHP runtime. A normal single `composer.lock` is fine when it works for the supported dependency graph. Use PHP-versioned lock files when old PHP support requires a different graph and supply-chain pinning is required. If the target project supports an older PHP runtime than any compatible Deptrac can run on, run Deptrac from a separate toolchain or PHAR on a newer PHP runtime.

Older Deptrac lines used for PHP 8.0 compatibility may not provide `vendor/bin/deptrac`. In that case, use a project-root launcher such as `deptrac.php` that prefers `vendor/bin/deptrac` and falls back to `vendor/deptrac/deptrac/deptrac.php`, then call it from Composer:

```json
{
    "scripts": {
        "deptrac": "@php deptrac.php analyse --config-file=deptrac.yaml"
    }
}
```

## Layer Discovery

Do not start from a fixed layer model. Discover boundaries from the target project:

1. Use Composer autoload roots to choose analysis paths.
2. Inspect real top-level directories and namespaces.
3. Keep only groups that have a responsibility boundary worth enforcing.
4. Define allowed dependency direction from project docs, entry points, and current dependency evidence.
5. Avoid suppressing violations during setup.

The skill includes example files under `skills/setup-toolkit-deptrac/examples/`. They are examples only, not templates to copy blindly:

- `application-layers.deptrac.yaml`
- `component-layers.deptrac.yaml`
- `library-public-api.deptrac.yaml`
- `module-boundaries.deptrac.yaml`
- `phpstan-extension-components.deptrac.yaml`

Use these examples only to see how discovered layers can be represented with collectors and rulesets.

## Verification

Run:

```bash
composer deptrac
php deptrac.php debug:unassigned --config-file=deptrac.yaml
php deptrac.php debug:unused --config-file=deptrac.yaml
```

Use `debug:unassigned` to check collector coverage and `debug:unused` to find stale rules. Fix unassigned production classes before treating the ruleset as complete.
