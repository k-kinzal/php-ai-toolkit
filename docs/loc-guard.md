# LocGuard Configuration

LocGuard configuration generated by the `/setup-toolkit-loc-guard` skill.

## Purpose

LocGuard is a first-party CLI for enforcing source metrics without PHPMD. It checks PHP files for physical file length, file NCLOC, class-like length, function length, method length, and cyclomatic complexity.

## Command

Run:

```bash
vendor/bin/loc-guard --config=loc.yaml
```

Exit codes:

- `0`: no violations
- `1`: metric violations found
- `2`: configuration or runtime error

## Configuration

Default `loc.yaml`:

```yaml
paths:
  - src

exclude: []

limits:
  max_file_lines: 500
  max_file_ncloc: 350
  max_class_lines: 400
  max_trait_lines: 300
  max_interface_lines: 200
  max_enum_lines: 200
  max_function_lines: 50
  max_method_lines: 50
  max_cyclomatic_complexity: 20

report:
  reporter: ai
  order_by:
    - path
    - line
    - rule
```

The limit value itself is allowed. A method with exactly 50 lines passes when `max_method_lines` is `50`; a method with 51 lines fails.

## Metrics

`max_file_lines` counts physical lines exactly as stored on disk.

`max_file_ncloc` counts non-comment lines of PHP code. Blank lines, comments, doc comments, and PHP open/close tags are excluded.

`max_class_lines`, `max_trait_lines`, `max_interface_lines`, and `max_enum_lines` count physical lines from the class-like declaration line through its closing brace.

## Analysis Scope

Use production source roots only. Do not include `tests/` by default; test method and fixture length are intentionally out of scope.

`exclude` entries are glob patterns relative to the config directory:

```yaml
exclude:
  - 'src/Generated/*'
```

## Complexity

LocGuard starts each function or method at complexity `1` and increments for branch points such as `if`, `elseif`, loops, `case`, `catch`, boolean operators, null coalescing, ternary branches, and `match` arms.

## Reporting

Configure the reporter with `report.reporter`:

- `ai`: structured text with remediation guidance for coding agents.
- `text`: concise human-readable output.
- `json`: machine-readable JSON for CI and tooling.

Override the configured reporter from the CLI:

```bash
vendor/bin/loc-guard --config=loc.yaml --reporter=json
vendor/bin/loc-guard --config=loc.yaml --format=text
```

Configure violation ordering with `report.order_by`. Supported fields are `path`, `line`, `rule`, `actual`, and `limit`.
